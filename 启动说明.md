# Ta 项目启动说明（云服务器 Linux）

本文档适用于当前服务器上的部署结果。

项目路径：`/myproject/Ta`
后端端口：`8003`
Systemd 服务名：`ta`

## 一、项目结构说明

- 项目根目录：`/myproject/Ta`
- 后端入口：`/myproject/Ta/backend/main.py`
- Python 虚拟环境：`/myproject/Ta/.venv`
- 前端构建目录：`/myproject/Ta/frontend/dist`
- 主配置文件：`/myproject/Ta/config.yaml`
- Systemd 服务：`/etc/systemd/system/ta.service`

## 二、使用前你需要填的配置

启动前，先编辑：

`/myproject/Ta/config.yaml`

重点检查这些配置项是否已经填好：

### 1）LLM 配置
至少要确认：
- `llm.provider`
- `llm.api_base`
- `llm.api_key`
- `llm.model`

### 2）管理员配置
至少确认：
- `admin.api_key`

### 3）如果你要启用这些功能，也要补对应 key
- TTS：`tts.*`
- ASR：`asr.*`
- 图像生成：`image_generation.*`
- 视觉识别：`vision.*`
- 语音网关：`voice_gateway.*`
- 其他第三方服务相关 key

注意：如果某些模块没准备好，建议先关闭对应 `enabled: true` 的模块，避免启动后报错。

## 三、启动方式

### 方式 A：通过 systemd 启动（推荐）

启动服务：
```bash
sudo systemctl start ta
```

重启服务：
```bash
sudo systemctl restart ta
```

停止服务：
```bash
sudo systemctl stop ta
```

查看运行状态：
```bash
systemctl status ta --no-pager
```

设置开机自启：
```bash
sudo systemctl enable ta
```

## 四、前端重新构建

如果你改了前端代码，需要重新打包：

```bash
cd /myproject/Ta/frontend
npm install
npm run build
```

打包完成后，后端会自动从以下目录提供前端静态文件：
`/myproject/Ta/frontend/dist`

一般只改配置文件、不改前端代码时，不需要重新 build。

## 五、后端依赖维护

如果你改了 Python 依赖，可在项目根目录执行：

```bash
cd /myproject/Ta
source .venv/bin/activate
pip install -r requirements.txt
```

## 六、常用排查命令

### 1）看后端日志
```bash
journalctl -u ta -n 100 --no-pager
```

实时看日志：
```bash
journalctl -u ta -f
```

### 2）检查端口监听
检查后端 8003：
```bash
ss -ltnp '( sport = :8003 )'
```

### 3）本机接口测试
测试后端直连：
```bash
curl http://127.0.0.1:8003/api/health
```

测试前端页面：
```bash
curl -I -H "Accept: text/html" http://127.0.0.1:8003/
```

## 七、访问方式

后端同时提供 API 和前端页面，无需 Nginx。

如果服务器安全组和防火墙已放行 8003 端口：

浏览器访问：
```text
http://你的服务器公网IP:8003/
```

API 健康检查：
```text
http://你的服务器公网IP:8003/api/health
```

## 八、配置改完后的标准操作

每次你修改 `config.yaml` 后，建议执行：

```bash
sudo systemctl restart ta
systemctl status ta --no-pager
```

如果你修改了前端代码，再执行：

```bash
cd /myproject/Ta/frontend
npm run build
sudo systemctl restart ta
```

## 九、当前部署要点总结

当前这套部署是：
- Ta 项目安装在 `/myproject/Ta`
- 后端通过 `systemd` 运行，服务名 `ta`
- 后端监听 `0.0.0.0:8003`
- 后端同时提供 API 接口和前端静态文件服务
- 浏览器直接通过 `http://IP:8003` 访问，无需 Nginx
- 根路径 `/` 浏览器访问自动返回前端页面
